# 产品项目空间

这个目录用于沉淀产品项目的长期上下文，避免每次对话都从零开始。

## 使用方式

1. 先维护 `context/product-context.md`，说明产品背景、用户、目标和范围。
2. 每次有重要决策，更新 `decisions/decision-log.md`。
3. 每次有用户反馈、数据或假设，更新 `evidence/evidence-table.md`。
4. 每次进入新工作流，更新 `workflows/workflow-state.md`。
5. 准备交给 UI 或研发时，更新 `handoffs/downstream-readiness.md`。

## 原则

- 不清楚的内容写“待确认”，不要编造。
- 重要假设必须显式记录。
- 下游交接前必须做门禁检查。
