# Post-Launch Learning

Close the product loop after launch.

## Required References

Read `../../references/chinese-output-standard.md` first. Read `../../references/evidence-taxonomy.md` and `../../references/gate-system.md`.

## Supporting PM Skills

Use available method skills when useful: `ab-test-analysis`, `cohort-analysis`, `metrics-dashboard`, `sentiment-analysis`, `retro`, `release-notes`, and `strategy-red-team`.

## Workflow

1. Restate launch expectations and success metrics.
2. Compare expected vs actual results.
3. Classify evidence and feedback.
4. Identify learning, surprises, and data gaps.
5. Decide keep, iterate, roll back, or investigate.
6. Route follow-up work to the next scene.

## Output

Use `../../assets/templates/post-launch-review.md` for the artifact.

Always include:

- Result summary.
- Evidence table.
- Decision.
- Follow-up owner.
- Next PM Superpowers workflow.

## Rules

- Do not call a launch successful without linking back to the original metric or user outcome.
- If measurement is missing, mark the learning gate `BLOCKED` and propose instrumentation.
