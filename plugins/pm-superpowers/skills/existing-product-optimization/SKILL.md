# Existing Product Optimization

Diagnose and improve existing product performance.

## Required References

Read `../../references/chinese-output-standard.md` first. Read `../../references/scene-taxonomy.md`, `../../references/evidence-taxonomy.md`, `../../references/gate-system.md`, and `../../references/supporting-pm-skills.md`.

## Supporting PM Skills

Use available method skills when useful: `customer-journey-map`, `north-star-metric`, `metrics-dashboard`, `cohort-analysis`, `brainstorm-ideas-existing`, `brainstorm-experiments-existing`, `pre-mortem`, and `test-scenarios`.

## Workflow

1. Define the current feature, user flow, and affected segment.
2. Identify the symptom, baseline metric, and user pain.
3. Separate evidence from assumptions.
4. Diagnose likely causes across product, UX, data, policy, and operations.
5. Generate solution options and experiments.
6. Prioritize by expected outcome, effort, and risk.
7. Run metric and solution gates before PRD or sprint planning.

## Block Conditions

Block when there is no defined symptom, affected segment, or observable baseline. If data is missing, propose instrumentation before optimization.
