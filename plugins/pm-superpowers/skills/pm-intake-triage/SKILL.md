# PM Intake Triage

Clarify incomplete PM requests and turn them into a usable product brief.

## Required References

Read `../../references/chinese-output-standard.md` first. Read `../../references/scene-taxonomy.md`, `../../references/gate-system.md`, and `../../references/evidence-taxonomy.md`.

## Workflow

1. Identify what the user is trying to accomplish.
2. Separate known facts, assumptions, opinions, and decisions needed.
3. Ask only the minimum clarifying questions needed to select a workflow.
4. If enough context exists, produce a brief and route to the next PM Superpowers skill.
5. If not enough context exists, mark the work `BLOCKED` and state exactly what is missing.

## Intake Questions

Use these only when missing:

- What product, user, or business area is this about?
- What decision should this work support?
- Who is the target user or customer segment?
- What evidence or feedback exists?
- What output do you need next?
- Who will consume the output: PM, design, engineering, sales, leadership, or operations?

## Output

Use `../../assets/templates/product-brief.md` when creating a reusable brief.

Always include:

- Clarified request.
- Evidence vs assumptions.
- Recommended scene and next skill.
- Gate status.
- Missing inputs, if any.
