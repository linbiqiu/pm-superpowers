# Feedback To Roadmap

Turn messy feedback into roadmap-quality decisions.

## Required References

Read `../../references/chinese-output-standard.md` first. Read `../../references/scene-taxonomy.md`, `../../references/evidence-taxonomy.md`, `../../references/gate-system.md`, and `../../references/supporting-pm-skills.md`.

## Supporting PM Skills

Use available method skills when useful: `analyze-feature-requests`, `sentiment-analysis`, `prioritize-features`, `outcome-roadmap`, `stakeholder-map`, and `customer-journey-map`.

## Workflow

1. Normalize feedback into source, user segment, request, pain, and urgency.
2. Classify evidence type and confidence.
3. Cluster feedback into themes and opportunities.
4. Separate solution requests from underlying user problems.
5. Score opportunities by user impact, business impact, frequency, and risk.
6. Convert selected opportunities into roadmap candidates.
7. Gate before committing roadmap items.

## Block Conditions

Block when feedback has no source, no user segment, or no evidence of frequency/severity and the requested decision is roadmap commitment.
