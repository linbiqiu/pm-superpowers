# Metrics Experiments

Turn product questions into measurable decisions.

## Required References

Read `../../references/chinese-output-standard.md` first. Read `../../references/evidence-taxonomy.md`, `../../references/gate-system.md`, and `../../references/supporting-pm-skills.md`.

## Supporting PM Skills

Use available method skills when useful: `north-star-metric`, `metrics-dashboard`, `sql-queries`, `ab-test-analysis`, `cohort-analysis`, `dummy-dataset`, and `brainstorm-experiments-existing`.

## Workflow

1. State the product decision and hypothesis.
2. Define primary, secondary, and guardrail metrics.
3. Check whether tracking and data sources exist.
4. Choose analysis, dashboard, cohort, or experiment path.
5. Define decision thresholds.
6. Produce the metrics or experiment plan.
7. Gate before launch or readout.

## Block Conditions

Block when a metric is requested without a decision, data source, or measurable behavior. If data is unavailable, produce an instrumentation request instead of analysis.
