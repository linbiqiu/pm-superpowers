# Prioritization Roadmap

Convert candidate work into a defensible outcome roadmap.

## Required References

Read `../../references/chinese-output-standard.md` first. Read `../../references/scene-taxonomy.md`, `../../references/gate-system.md`, `../../references/evidence-taxonomy.md`, and `../../references/supporting-pm-skills.md`.

## Supporting PM Skills

Use available method skills when useful: `prioritization-frameworks`, `prioritize-features`, `outcome-roadmap`, `brainstorm-okrs`, `stakeholder-map`, and `strategy-red-team`.

## Workflow

1. Clarify decision horizon and constraints.
2. Normalize candidate items into comparable units.
3. Select a prioritization method appropriate to the decision.
4. Score or rank with visible assumptions.
5. Translate priorities into Now, Next, Later or an outcome roadmap.
6. Record tradeoffs and dropped options.
7. Run gate review before treating the roadmap as committed.
